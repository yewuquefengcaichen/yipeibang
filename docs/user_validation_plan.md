# 用户验证计划

## 目标用户

- 慢病患者：糖尿病、高血压、血脂异常、甲状腺复查等人群
- 陪诊家属：需要整理材料、同步医生交代、避免遗漏的人群
- 学生或年轻用户：需要帮助同学或家人做初步就医信息整理

## 验证任务

| 任务 | 观察指标 |
| --- | --- |
| 输入夜间出汗问题 | 是否能理解不是直接诊断，而是风险分层和记录建议 |
| 输入血糖 8.2 复诊 | 是否能生成可带去医院的材料清单和问诊问题 |
| 输入血压 150/96 | 是否能自动保存记录并提示复查 |
| 创建提醒 | 是否能看懂提醒来源、清单和完成状态 |
| 浏览知识库 | 是否能通过症状、疾病、指标、用药、生活方式找到内容 |
| 急症输入 | 是否明确建议 120/急诊，没有给危险用药建议 |

## 反馈收集

- 页面是否显得可信、专业、不过度花哨
- 回答是否具体，是否有“下一步该做什么”
- 记录、提醒、档案是否像真实产品，而不是静态展示
- 移动端输入和按钮是否容易点击

## 迭代规则

- 用户说“看不懂”：减少术语，增加清单化表达
- 用户说“太慢”：优先保持本地首字和 thinking，不等待外部 API
- 用户说“没用”：检查该按钮是否真实写入或跳转
- 用户说“不安全”：扩大 SAFE 规则和用药边界
