# 医陪帮 3 分钟期末汇报演示脚本

## 0:00-0:30 目标用户与痛点

医陪帮面向老年慢病患者和陪诊家属。典型用户是 68 岁糖尿病患者陈阿姨：复诊材料容易漏带，检查报告看不懂，低血糖等风险不知道怎么问医生，家属也经常需要反复沟通。普通系统需要点菜单、填表和自己整理材料，医陪帮的目标是让用户用自然语言说一句话，系统直接完成识别、整理、记录和提醒。

## 0:30-2:00 核心功能现场演示

打开首页，点击“一键跑核心演示”或输入：

```text
明天内分泌科复诊，空腹血糖8.2，最近夜里出汗，要带什么？
```

演示重点：

- 中间对话区出现流式输出，像真实智能体一样逐步生成。
- 右侧“意图识别与路由”显示 HYBRID，说明同时识别复诊准备、报告解读、风险关注和主动提醒。
- “知识库命中”显示空腹血糖、糖化血红蛋白、复诊材料等 RAG 依据。
- “记忆更新”写入常用科室、血糖记录、夜间出汗、提醒偏好等。
- “系统执行动作”生成复诊材料清单、问诊问题提纲、复诊提醒和家属摘要。

然后演示安全边界，输入：

```text
我胸痛、呼吸困难、脸色发白，现在怎么办？
```

演示重点：

- 路由切换为 SAFE。
- 系统不继续普通问答，不给诊断，不给药物建议，而是提示立即急救。
- 这体现医疗场景的安全层和风险拦截。

## 2:00-2:30 差异化说明

与普通表单系统相比，医陪帮不要求老年用户找菜单、填复杂表格，而是通过自然语言触发操作。与通用问答助手相比，医陪帮不是“问一句答一句”，而是围绕慢病复诊形成数据闭环：意图识别、知识库检索、长期记忆、主动提醒、家属协同和安全拦截都能现场看到。

## 2:30-3:00 用户验证与迭代

切换到“答辩评分点”页面，说明项目按汇报要求组织：

- 目标用户不是泛泛人群，而是老年慢病患者和陪诊家属。
- 核心功能不是大模型套壳，而是聊天即操作。
- 病例样例库、知识库、记忆系统、主动服务都有可操作页面。
- 后续用户验证重点收集 3 名真实目标用户反馈，验证材料是否更少漏带、家属沟通是否减少、提醒是否有帮助。
